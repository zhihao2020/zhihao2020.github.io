得到真正的业务ID

  2020-04-23   数据分析   爬虫 0 0  评论 字数统计: 907字 阅读时长: 2分

对于爬虫来说，知道了网站的数据自增ID，那么就能将整个站点爬取下来。不过，那些大站，比如油管之类的，他们绝不会把业务的自增ID暴露出来，而是用一种可逆的字符串替代。

一般有这么几种方法:

  • 使用Hashids，Hashids是一个小型的开放源代码库，可从数字生成短的，唯一的，非顺序的ID。
  • Base64编码，这个方法最直接，大多数语言也具有这个功能。不过没有Hashids的花哨功能。
  • 根据timestamp生成ID，使用计数器（如果有的话）+时间戳+一些系统值+一个随机整数。许多公司都采用这种方法，因为它在分布式系统中效果很好。这些id是彼此独立生成的，发生碰撞的风险非常小，可以忽略不计。
  • 如果ID仅包含数字，可以使用Optimus，它基于Knuth的整数哈希方法，并产生混淆的整数ID。只有PHP和Go实现。
  • Nano ID，一个很小的，安全的，URL友好的，唯一的JavaScript字符串ID生成器。它也可用于多种语言。
  • 其他的方法

使用hashids

安装

pip install hashids

使用

首先初始化一个hashids示例

from hashids import Hashids

编码（加密）

id = hashids.encode(2)

生成的字符ID长度和数值大小有关。

解码（解密）

num = hashids.decode('j0gW')

解码后返回时一个元组。如果你解码一个随机的字符串ID，不一定能恢复成数字，此时会返回空元组。

不过，因为hashids可逆，所以拿到ID后可以反向推测出原始数字ID。我们在编码的时候可以指定盐值。理论上，盐值不泄露，就很难破解。

指定盐值

hashids = Hashids(salt="aaa")
hashids.encode(123)

同样的数字使用不同的盐值编码得到的字符串是不同的，别人就无法猜出原始ID是多少了。

限制hash长度

默认情况下，数字越小，编码生成的字符串也比较断，为了讲不通大小的数值转化为统一长度的字符串，在初始化hashids时，可以指定hash的最小长度。

hashids = Hashids(salt="aaa",min_length=10)
hashids.encode(123)

限制hash的字符种类

如果你不希望hash中有大写字母，那么可以在初始化的时候，指定hash字符串的范围，例如我们可以指定为所有的小写字母

hashids = Hashids(alphabet="abcdefghigklmopkrst")
hashids.encode(123)

需要注意的时候，alphabet至少要指定16个以上的不同字符。

Base64

Base64是一种基于64个可打印字符来表示二进制数据的表示方法。由于

$$ \log_264=6 $$ 所以每6个比特为一个单元，可以对应某个打印字符。
